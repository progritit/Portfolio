## 🎨 Project Overview & Design Identity

The goal of this project was to build a fully responsive, professional portfolio website to anchor my CV, LinkedIn, and GitHub profiles.

Rather than using a generic template, I wanted the design to express a personal visual direction: a **Cyber-Solar interface identity**.

This design direction combines:

- Deep dark-mode backgrounds inspired by cyber interfaces
- Solar gold accents and energetic highlights
- Soft green details connected to solarpunk and organic design
- Subtle cosmic/orbital elements
- Clean, structured layouts focused on readability and usability

The portfolio is designed to feel bold, calm, purposeful, and technically grounded. It reflects my interest in building digital solutions that are not only visually engaging, but also accessible, responsive, scalable, secure, and thoughtfully designed.

To strengthen my front-end fundamentals, the website was intentionally built using **vanilla web technologies without frameworks**.

## 💡 What I Learned

Building a complete portfolio without the safety net of frameworks was an important exercise in core front-end engineering. My biggest takeaways include:

*   **Turning Learning into Production:** This portfolio helped me move from isolated exercises to a real, published project that presents my work, identity, and learning journey in one place.

*   **Deepening DOM Control:** Managing the mobile navigation and interactive elements manually with vanilla JavaScript gave me a stronger understanding of how the browser responds to user actions.

*   **Clean CSS Architecture:** Using CSS variables, reusable cards, responsive grids, and consistent spacing allowed me to maintain a specific cyber-solar visual identity without letting the stylesheet become difficult to manage.

*   **Responsive Design Thinking:** Adapting sections, project cards, navigation, and typography for different screen sizes helped me think more intentionally about mobile-first and user-friendly layouts.

*   **Production Workflows:** Moving from local development to GitHub Pages helped me practice Git workflow, deployment readiness, asset paths, repository organization, and incremental commits.

*   **Portfolio Positioning:** Replacing placeholders with real projects helped me understand how to communicate not only what I built, but also what each project helped me learn.
